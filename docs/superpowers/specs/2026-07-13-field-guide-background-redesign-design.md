# 生きもの図鑑背景リデザイン

## 結論

PC表示のコンテンツ幅外だけを、A案「生きもの図鑑」の背景へ置き換える。幅480px内のログイン画面、施設一覧、施設詳細、フォーム、文言、機能は変更しない。

## 成功条件

- PC表示を見た時、背景だけで動物園と水族館の両方を連想できる
- 背景は観察記録・生きもの図鑑の雰囲気を持ち、中央コンテンツより目立たない
- スクロールしても背景の位置が動かない
- 480px以下では背景装飾を表示せず、既存のスマホ表示を維持する
- 横スクロールを発生させない

## ビジュアル

背景色は生成りの図鑑用紙を思わせる #ece6d7 とする。22px間隔の薄い方眼を重ね、紙面の情報量は中央コンテンツより低く抑える。

左側には低彩度の緑 #315849 でキリンと植物の線画を置き、縦向きの分類ラベル「ZOOLOGY / FIELD 01」を添える。右側には青緑 #267080 で魚と水中生物の線画を置き、「AQUATIC LIFE / FIELD 02」を添える。

線画は塗りを使わず、細い輪郭線を基本とする。常時アニメーション、写真、絵文字、チケットやスタンプのコラージュは追加しない。

## 実装境界

- src/styles.css: 生成りの固定背景、方眼、左右の配置、分類ラベル、レスポンシブ制御
- src/assets/zoo-field-guide.svg: キリンと植物の線画
- src/assets/aquarium-field-guide.svg: 魚と水中生物の線画
- src/styles.test.ts: 背景色、SVG参照、固定指定、480px以下の非表示を検証

SVGをCSSのdata URIへ埋め込まず、独立ファイルにする。線画を差し替える時に長いCSSエスケープを編集せずに済むためである。

Reactコンポーネント、Firebase、Firestore、施設データ、訪問記録CRUDには触れない。

## レスポンシブ

- 481px以上: 中央480pxの外側に図鑑背景を表示する
- 480px以下: 左右の線画と分類ラベルを非表示にする
- 中央コンテンツ: width: min(100%, 480px) を維持する
- 狭いPC幅では余白に収まる範囲だけ線画を見せ、中央コンテンツの下へ透けさせない

## 検証

- CSS回帰テストで生成り背景、SVG参照、background-attachment: fixed、モバイル非表示を確認する
- PC幅1200pxで左右の線画と分類ラベル、中央480px配置、横スクロールなしを確認する
- スクロール前後で背景と線画の位置が変わらないことを確認する
- スマホ幅390pxで線画非表示、コンテンツ幅390px、横スクロールなしを確認する
- 既存テスト、lint、build、施設データ検証を通す

## 将来の破綻要因

線画を増やし続けると中央コンテンツとの主従が逆転する。左右それぞれの主要線画は1点に限定し、追加装飾は分類ラベルと方眼だけに留める。
