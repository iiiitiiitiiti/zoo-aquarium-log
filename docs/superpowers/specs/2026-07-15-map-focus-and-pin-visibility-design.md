# 地図ピン視認性・施設フォーカス改善 設計

## Goal

訪問済みピンを地図上で見分けやすくし、地図と施設詳細の往復で対象施設の位置を失わないようにする。また、施設詳細から地図を開いた場合は対象施設だけを表示する。

## Requirements

1. 訪問済みピンは現在の暗い緑から、明るい緑の本体色と濃い緑の縁取りへ変更する。凡例も同じ配色にする。
2. 地図上の施設から詳細へ移動して「地図に戻る」を押した場合、対象施設を中心にズームした地図を再表示する。
3. 施設詳細の「地図で場所を見る」から地図を開いた場合、対象施設1件だけを表示し、そのピンのポップアップを開く。
4. 既存の検索・種別・都道府県・営業状態・訪問状態フィルター、マーク表示、ポップアップからの詳細遷移を維持する。

## Design

`App` が地図の表示モードを `all` と `facility` に分けて管理する。地図から詳細へ移るときは対象施設IDを復元用フォーカスとして保持し、戻るときは全施設を表示したまま対象施設を中心にする。詳細から地図を開くときは表示対象をその施設1件に絞り、同じ施設IDをフォーカスとして渡す。

`MapPanel` は既存の `focusedFacilityId` を利用し、初期表示時に対象施設を `setView(..., 12)` してポップアップを開く。表示対象が空の場合は既存の日本全域表示を維持する。地図を閉じるときはフォーカスを解除して次回の通常表示へ戻す。

ピン配色は `mapPins.ts` に集約し、訪問済み本体を `#35b978`、縁取りを `#0f6b46` とする。未訪問のテラコッタとお気に入り・行きたいバッジの配色は変更しない。

## Error and edge-case handling

- 対象施設IDが現在のフィルター結果に存在しない場合、地図は通常の表示対象を使い、既存の初期表示ロジックへ戻る。
- 施設詳細から地図を開いた状態で「施設一覧」に戻った場合、フォーカスと1件表示モードを解除する。
- 地図から詳細へ移動した後、連続クリックや詳細からの再遷移があっても、最後に選択した施設IDをフォーカスとして扱う。
- 表示施設0件、訪問記録未ロード、マーク未ロードでは既存のローディング・空状態・表示制御を変えない。

## Verification

- `mapPins` の配色・凡例テストで訪問済み色と縁取りを固定する。
- `MapPanel` のフォーカス表示テストで対象施設への `setView` とポップアップ開放を確認する。
- `App` のテストで、地図→詳細→地図のフォーカス復元と、詳細→地図の1件表示を確認する。
- `npm test -- --run`、`npm run lint`、`npm run build`、`git diff --check` を実行する。
