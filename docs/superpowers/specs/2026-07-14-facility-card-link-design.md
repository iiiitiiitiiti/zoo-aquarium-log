# 施設カードの詳細リンク化

## 結論

施設一覧の `.facility-card` 全体を、既存の施設詳細表示へ進む内部リンクにする。カード内の「記録を見る」ボタンと公式サイトリンクは一覧から外し、公式サイトへのリンクは施設詳細ページの `VisitPanel` に集約する。

## 方針

- 既存の `selectedFacility` 状態による画面切り替えを維持し、ルーターは導入しない。
- カードは `a.facility-card` として描画し、クリック時に既定のハッシュ遷移を抑止して `selectedFacility` を設定する。
- カードの `href` には施設 ID を含め、リンクとしての対象を明確にする。
- カード内に別のボタンやリンクを置かず、入れ子のインタラクティブ要素を避ける。
- 詳細ページの公式サイトリンクは既存の `VisitPanel` のリンクを利用する。

## テスト

- 施設カードがリンクとして表示され、施設 ID を含む `href` を持つことを確認する。
- カードをクリックすると施設詳細ページへ切り替わることを確認する。
- 一覧画面では公式サイトリンクが表示されず、詳細画面では公式サイトリンクが表示されることを確認する。
- 既存の検索、絞り込み、訪問記録画面の戻る操作を維持する。
