# 動物園・水族館ログ 初期公開版デザイン仕様

## 目的

家族がスマートフォンで全国の動物園・水族館を探し、今後の訪問記録につなげられる初期公開版を作る。Firebase連携前のため、今回はパイロット施設マスタの検索・絞り込み・閲覧に範囲を限定する。

## 表示領域

- アプリ本体はスマートフォン向けの1カラムとする。
- PCでもコンテンツ幅を最大480pxに固定し、画面中央へ配置する。
- 480pxより狭い端末では画面幅いっぱいに表示する。
- PCの余白は森林を思わせる静かな背景とし、別のPCレイアウトには展開しない。

## ビジュアル

- コンセプトは「家族のフィールドノート」。自然観察の記録帳と水辺の案内標識を組み合わせる。
- 色は forest `#173F35`、leaf `#2F6B50`、water `#2A7180`、sky `#DCEFF0`、paper `#F7F4E8`、ink `#18231F` を使う。
- 見出しは日本語の丸みがある太字ゴシック、本文は可読性の高いシステムゴシックを使い、外部フォント依存を避ける。
- 記憶に残る要素は、ヘッダーから施設一覧へ伸びる「探索ルート」の細い曲線。装飾はここに集中させる。

## 操作と状態

- 名称・読み仮名・都道府県・市区町村を対象に部分一致検索する。
- 「すべて」「動物園」「水族館」「複合・その他」で種別を絞り込む。
- 結果件数を常時表示する。
- 0件の場合は条件を戻すよう案内する。
- 施設カードの公式サイトリンクは新しいタブで開く。
- キーボードフォーカスを明示し、`prefers-reduced-motion` では動きを止める。

## データとエラー

- `src/data/facilities.json` を静的に読み込む。
- ビルド前テストで件数、必須項目、ID重複、列挙値、座標、URL、確認日を検証する。
- 初期版に訪問済み等の未保存状態を見せず、未実装機能はUIに置かない。

## 公開構成

- Vite + React + TypeScriptを使用する。
- Viteの `base` は `/zoo-aquarium-log/` とする。
- GitHub Actionsで `main` のビルド成果物をGitHub Pagesへ公開する。
- CIでは施設データテスト、TypeScriptビルドを実行する。

## 検証

- 施設検証テストがすべて成功すること。
- 本番ビルドが警告・エラーなく完了すること。
- スマホ幅とPC幅で、アプリ面が最大480pxに保たれることを目視確認する。
- 検索、種別フィルター、0件表示、公式リンク、キーボード操作を確認する。
