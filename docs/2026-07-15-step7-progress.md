## 次回再開時の手順（北海道ブロックの仕上げ = チャンク④から）

1. **Phase R（独立レビュー）**: 調査班とは別の Sonnet エージェントで再確認
   - **100%再確認（2件）**: ノースサファリサッポロ（closed 判定の再裏取り）、オホーツクエミューらんど（調査時に公式サイトがメンテ中で一次情報未達。sourceUrls も二次情報のみ）
   - **層化サンプリング（4件程度）**: 残り23件から班・type・地域で層化抽出し一次情報を再調査・突合。不一致率が10%超なら該当班の全件再確認
2. **Phase D（中央マージ）**: メインで実施
   - proposedId の表記揺れ統一・確定（bear_park/bear_ranch 等の訳語は公式英語名優先で正規化）
   - **調査結果の note を facilities.json の `note` フィールドへ引き継ぐ**（2026-07-15 ユーザー決定でスキーマに任意フィールド note を追加済み・VisitPanel に表示済み）。対象は変則休園・季節営業・整備休館などの来訪計画に影響する情報のみ。1〜500文字・「2026年時点」等の確認時点を含めて要約する。単なる調査メモ（座標根拠等）は含めない
   - `npm run check:duplicates` → 既存150件との重複最終確認
   - facilities.json へ都道府県コード順（北海道は先頭）・県内かな順で挿入（150→175件）
   - `npm run test:data` ＋ `node scripts/check-geo.mjs` → 警告の個別解消
   - コミット例: `feat: 施設マスタに北海道の非加盟施設25件を追加（150→175件）`
3. 以降は東北ブロック（new 12件）から同じ刻み（調査2班/チャンク→ブロック仕上げ）で進行
