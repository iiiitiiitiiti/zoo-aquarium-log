# 施設メモ複数化とMarkdown表示 設計書

## 目的

施設詳細ページの施設メモを、1施設1件の上書きメモから複数件の個別メモへ変更する。各メモを編集・削除できるようにし、訪問記録の「メモ・感想」と施設メモの保存後表示をMarkdown対応にする。

## 確定した要件

- 施設メモにタイトルは持たせない。
- 施設メモは1施設に複数件追加できる。
- 各施設メモに編集・削除操作を付ける。
- 更新日は現在より小さく、薄く表示する。
- Markdownは入力中にプレビューせず、保存後の表示時だけHTMLへ変換する。
- 訪問記録の既存入力データは原文のまま保持し、表示時にMarkdownとして扱う。
- 既存の1施設1メモ形式のFirestoreデータは読み続けられるようにする。
- 既存の2000文字上限を施設メモ・訪問メモの双方で維持する。

## 採用する構成

### 1. 施設メモのデータモデル

既存の `households/{householdUid}/facilityNotes` コレクションを継続利用する。新規メモは自動生成IDのドキュメントとして保存する。

```text
households/{householdUid}/facilityNotes/{noteId}
{
  facilityId: string,
  text: string,
  createdAt: timestamp,
  updatedAt: timestamp
}
```

旧形式のドキュメント（ドキュメントIDが施設IDで、`facilityId` と `createdAt` がないもの）は、購読時にドキュメントIDを施設IDとして補完する。編集時に `facilityId` を補完して新形式へ寄せるが、強制移行は行わない。

### 2. Markdownレンダリング

`marked`でMarkdownをHTMLへ変換し、`dompurify`でサニタイズした結果だけを表示する。raw HTML、`javascript:` URL、スクリプトなどは表示しない。

共通の `renderMarkdown` 関数を用い、訪問メモと施設メモの表示仕様を揃える。textareaには常にMarkdown原文を表示する。

### 3. UI

施設メモはカードのリストとして表示する。メモ追加時・編集時は既存の保存ボタン方式を維持し、同時に複数フォームは開かない。削除時は訪問記録と同様に確認ダイアログを表示する。

表示順は `updatedAt` の新しい順を基本とし、旧形式や未確定タイムスタンプは後方へ置く。更新日は `small` 相当の文字サイズと低コントラストで表示する。

### 4. エクスポート

`facilityNotes` は配列のまま維持し、各項目に `id`、`facilityId`、`text`、`createdAt`、`updatedAt` を含める。複数メモを表現できるようエクスポートスキーマをv3へ更新する。既存v2の読み込み機能は現状存在しないため、今回の対象外とする。

### 5. Firestoreルール

施設メモの許可フィールドに `facilityId` と `createdAt` を追加し、施設ID・本文長・作成日時・更新日時を検証する。旧形式の読み取りと削除は継続して許可し、新規作成・更新は新形式を要求する。

## 代替案と不採用理由

- 施設ごとにサブコレクションを追加する案: 新しいデータモデルは明快だが、既存ドキュメントの移行が必要になるため採用しない。
- 独自Markdownパーサーを実装する案: 依存関係は増えないが、リンクや日本語Markdownの端ケースと安全対策を自前で維持する必要があるため採用しない。
- WYSIWYGまたは入力中プレビュー: 視覚的には便利だが、今回の要望はMarkdown入力であり、入力UIと依存関係が大きくなるため採用しない。

## エラー・境界条件

- 空文字保存は新規作成せず、既存メモの編集では削除扱いにする。
- 2000文字超過は保存前に拒否する。
- 連打中は保存中の操作を無効化する。
- メモ購読失敗時は編集・追加を無効化し、既存内容の表示は維持する。
- Markdownが空、壊れたリンク、危険なHTMLを含む場合もサニタイズ後に安全に表示する。
- 絵文字・日本語・改行をそのまま保持する。

## 成功条件

- 1施設に複数の施設メモを作成でき、各メモを編集・削除できる。
- 既存施設メモが表示・編集できる。
- 訪問メモと施設メモで見出し・箇条書き・太字・リンクなどが安全に表示される。
- JSONエクスポートで複数メモが失われない。
- Firestoreルール、テスト、lint、build、デプロイが成功する。
