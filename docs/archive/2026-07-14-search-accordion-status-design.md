# 施設検索アコーディオン・営業状態フィルター設計

## 目的

検索、種別、都道府県、営業状態の操作が常時表示されることで施設一覧が縦に伸びるため、検索操作を必要なときだけ開ける構成にする。同時に、施設カードの営業状態を実データに合わせ、営業状態でも絞り込めるようにする。

## 採用設計

- 検索領域は標準の `details` / `summary` を使い、初期状態を閉じる。
- 検索キーワード、種別、都道府県、営業状態を検索領域内へまとめる。
- 営業状態は既存のデータ定義に合わせて「営業中」「休園中」「閉園済み」の3値で扱う。
- 絞り込み条件はアコーディオンを閉じても保持し、summaryに適用中の条件数を表示する。
- 施設カードの営業状態タグは `open / suspended / closed` の値から表示ラベルと色を決める。

## 不採用案

- JSで開閉状態を持つ専用ボタン: `details` の標準キーボード操作と開閉セマンティクスを再実装する必要があるため不採用。
- モーダルやドロワー: 施設一覧から視線が離れ、今回の「常時表示によるゴチャつき」以上の操作負荷を生むため不採用。

## エッジケース

- 条件が未指定のときは全施設を表示する。
- 条件を複数指定したときは、既存の検索・種別・都道府県フィルターと営業状態をANDで適用する。
- `suspended` と `closed` はデータ上の意味が異なるため、表示上も別タグにする。
- アコーディオンを閉じても、入力値・選択値・結果件数は変更しない。

## 検証

- `filterFacilities` の営業状態絞り込みテスト
- 初期閉状態、開閉、営業状態タグ、実データに基づくカード表示のUIテスト
- 既存のデータ検証、全テスト、lint、build、公開デプロイの成功確認
