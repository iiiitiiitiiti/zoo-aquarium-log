# 詳細画面ヒーロー操作整理 設計

## 目的

施設詳細画面の `.detail-hero` 内で、訪問記録ボタンの位置は変更せず、施設情報への導線と個人用マークの4操作を役割ごとに整理する。

## 対象ユーザーと成功条件

- 対象: スマートフォンを中心に動物園・水族館の訪問記録を残すユーザー
- 成功条件: 「地図で場所を見る」「公式サイトを見る」を施設情報グループに、「行きたい」「お気に入り」を個人用マークグループに分け、各操作が成り行き幅のまま見つけやすく並ぶ

## 採用する構成

```text
[ 地図で場所を見る ] [ 公式サイトを見る ↗ ]

[ ♡ 行きたい ] [ ★ お気に入り ]
```

- 訪問記録の追加ボタンは既存の訪問記録セクション見出し横に残す
- 地図ボタンと公式サイトリンクを `.detail-hero-links` にまとめ、flex-wrap とgapで横並びにする
- 行きたい・お気に入りは既存の `.mark-toggles` を独立した次の行として維持する
- 4操作は固定幅にせず、内容に応じた成り行き幅とする
- 画面幅が狭い場合は自然に折り返す
- 手動追加施設の編集・削除操作は4操作と混ぜず、既存の施設操作グループに残す

## 変更しないこと

- 操作のイベント処理、Firestore保存、ルーティング
- 詳細画面上部の戻るボタン
- 訪問記録セクションおよび訪問記録追加ボタン
- 選択中のマーク表示

## 実装方針

- `src/VisitPanel.tsx` の4操作にグループ用ラッパーと公式サイト用クラスを追加する
- `src/styles.css` に施設情報リンク行のflexレイアウトを追加する
- 既存の色、角丸、フォント、選択状態を流用し、視覚的な階層だけを整理する
- 小規模なレイアウト変更のため、テストスイートは実行せず `npm run lint` と差分確認で検証する
