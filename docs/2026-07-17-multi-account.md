# 複数世帯アカウント対応の実装状況

2026-07-17時点で、複数世帯に対応するためのアプリ側基盤を実装済みです。各世帯はFirebase AuthのUIDを境界として扱い、世帯ごとに別のFirestore・Storage Storeを生成します。

## 現在の状態

- 既存の「家族」アカウントを設定済みアカウントとして維持
- 世帯B・世帯Cは実ユーザーのUIDとメールアドレスが未確定のため、準備中表示のプレースホルダーを配置
- ログイン前後で共通の世帯選択UIを使用
- ログイン後は別世帯の合言葉を入力して切り替え可能
- 一度認証に成功した世帯の合言葉は、UID単位でlocalStorageに保存して次回の切り替えに利用
- 保存済みの合言葉は世帯単位で削除可能。削除しても現在のログイン状態は終了しない
- 切り替え時はUID単位でStoreを作り直し、前の世帯の表示・購読状態を引き継がない

## 注意点

localStorageに保存する合言葉は平文です。共有端末・紛失端末・ブラウザの開発者ツールから読み取られる可能性があるため、信頼できる端末だけで記憶機能を使用してください。リポジトリやエクスポートJSONには合言葉を保存しません。

現在は追加世帯の実UIDが未確定のため、Firestore RulesとStorage Rulesの許可リストは既存UIDのみです。準備中のUIDを本番ルールへ追加しないでください。

## 後工程：世帯B・世帯Cを有効化する手順

1. Firebase Consoleで追加ユーザーを作成し、各ユーザーのUID・メールアドレス・表示名を確定する
2. `src/accounts.ts` の世帯B・世帯Cを実値へ置き換え、`configured: true` にする
3. `firestore.rules` と `storage.rules` の許可リストへ同じUIDを追加する
4. Rulesの統合テストを実行し、各世帯の自データのみ読み書きできることを確認する
5. Rulesをデプロイし、3世帯それぞれで初回ログイン・切り替え・データ分離を実機確認する

追加ユーザーの作成とUID確定が終わるまでは、現行アカウントだけを本番利用します。
